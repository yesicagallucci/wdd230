Animations in CSS

Introduction

Keyframes

Keyframes specify values for animating properties at different points. They determine the behavior of one cycle, which may repeat zero or more times. Besides the animation properties you can use transition and transform properties within keyframes.

Template:

@keyframes name { from { porperties; } to { properties; } }

or

@keyframes name { 0% { porperties; } 100% { properties; } }

Animation-name

For the Keyframes to work you need to add the animation-name property to the element you want to naimate for the keyframe to work, otherwise it won't. You can use any name you want but it has to match the one in the keyframe.

Animation-duration

This property defines the duration of the animation cycle. In other words it will determine how long it will take the element to complete the animation. This property is set in seconds or miliseconds

animation-duration: 10s;

Animation-delay

The animation-delay property specifies when an animation starts, allowing it to begin after it is applied or appear to have started beforehand. It's also set in seconds or miliseconds.

animation-delay: 1s;

Animation-iteration-count

This property is set with a number, which will be the number of times the animation cycle will run. The value of this porperty will affect the outcome of the animation-direction property, because some direction values need more than one iteration to work.

animation-iteration-count: 4;

This property can also have the value of infinite for a non-stop animation (check the Animation-timing-function below)

Animation-direction

The property 'animation-direction' determines whether an animation should play in reverse or not. This applies to a single cycle or all cycles. When an animation plays in reverse, the timing functions are also reversed.

In the example below normal and reverse values have an even iteration-count while alternate and alternate-reverse have an odd iteration-count. this is to show where they end up once the animation ends. But, to show where they end up I had to set the animation-fill-mode property as well.

Normal is the default value.
Reverse will play the animation from end to start or from 100% to 0%
Alternate and alternate-reverse values require a value above 1 in the iteration-count property to work.
Alternate will play one iteration from 0 to 100% and viceversa.
Alternate-reverse will play it from 100% to 0% and viceversa.

animation-direction: normal/reverse/alternate/alternate-reverse;

Animation-fill-mode

The animation-fill-mode property determines the behavior of the element before and after an animation is executed.

animation-fill-mode: none/forwards/backwards/both;

none: no effect on the animation
forwards:the values applied will be those for the end of the last completed iteration of the animation, instead of the values for the start of the next iteration. This means that the element will stay with the values of the 100% if its direction is not reversed.
backwards:The elements will stay using the property values specified in the keyframe that marks the begining of the animation (0% if it's not reversed).
both: will apply both forwards and backwards values.

Animation Shorthand Property

All these animation-properties can be state using the following shorthand:
animation: name duration timing-function delay iteration-count direction fill-mode play-state

The values for animation-play-state are pause or running, it's just for pausing the animation, the default is running.
